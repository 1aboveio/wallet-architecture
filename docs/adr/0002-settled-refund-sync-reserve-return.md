# 0002 已结算交易退款时保证金同步退回

已结算交易发起退款时，保证金按退款比例同步退回商户可用余额，与退款在同一事务内完成。

## 背景

交易在 T+7 结算时会按比例冻结保证金（reserve）。如果交易在结算后发生退款，保证金的计算基础发生了变化——实际结算金额减少了，保证金也应该相应减少。

早期设计未明确这个场景的处理方式。两种方案：

- **方案 A（同步）**：退款和保证金退还在同一事务内完成
- **方案 B（分步）**：退款先扣 available，保证金在下一个结算周期统一调整

## 决策

选择方案 A：退款与保证金退回同步执行。

```
原交易 $100，已结算，保证金 $5
T+10 退款 $30

保证金调整:
  原保证金:          $5.00 ($100 × 5%)
  退款后结算金额:    $70 ($100 - $30)
  应有保证金:        $3.50 ($70 × 5%)
  应退回保证金:      $1.50 ($5.00 - $3.50)

记账分录（同一事务）:
  借  customer:abc:available:USD   -$30.00   (退款)
  贷  receivable:txn:USD           +$30.00

  借  customer:abc:reserve:USD     -$1.50    (保证金退回)
  贷  customer:abc:available:USD   +$1.50

净效果: available -$28.50, reserve -$1.50
```

## 理由

1. **商户体验清晰** — 一次操作看到完整影响，不会出现"退了款但保证金还没调"的困惑
2. **记账原子性** — 同一事务内完成，不存在中间状态
3. **退款校验一致** — 退款时的可用资金检查需要考虑保证金退回的净效果

## 影响

退款校验规则更新为：

```
退款净扣减 = 退款金额 - 保证金退回金额

IF 退款净扣减 > available_balance
  → REJECT "INSUFFICIENT_MERCHANT_FUNDS"
```
