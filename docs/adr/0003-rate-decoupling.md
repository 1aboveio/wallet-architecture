# 0003 保证金与手续费比率独立配置

保证金比率与 MDR（手续费比率）独立配置，不做交叉校验。当合计超过 100% 时，商户实收为负，记入 available 负余额，从后续收入抵扣。

## 背景

滚动保证金比率和 MDR 由不同团队/策略配置，存在合计超过 100% 的可能：

```
示例: 保证金 95%, MDR 10%, 合计 105%
商户每笔交易实收 = 100% - 95% - 10% = -5%
```

## 决策

不做配置级交叉校验，允许合计超 100%。差额记为商户 available 负余额。

### 理由

1. **配置解耦** — 保证金和 MDR 独立管理，不产生配置依赖
2. **灵活性** — 极高风险商户可能需要高保证金比例
3. **已有负余额机制** — 系统已支持负余额从后续收入抵扣

## 记账

```
交易 $100, 滚动保证金 95%, MDR 10%

  借  customer:abc:pending:USD          -$100.00
  贷  customer:abc:available:USD        -$5.00     ← 负余额
  贷  customer:abc:reserve:rolling:USD  +$95.00
  贷  revenue:fee:acquiring             +$10.00
```

## 后续收入抵扣

```
商户 available = -$5

后续交易 $1000, 保证金 50%, MDR 10%:
  应入 available = $1000 - $500 - $100 = $400
  先抵扣负余额 -$5:
  实际入 available = $400 - $5 = $395
```

## 运行时风控

虽然不做配置耦合，但需要运行时监控防止负余额失控：

```
规则 1: 连续负余额告警
  IF 商户 available_balance 连续 N 笔交易为负
    → 告警
    → 冻结新交易

规则 2: 负余额上限
  IF 商户 available_balance < -阈值
    → 冻结出金
    → 要求商户补充保证金（充值到 special_account → fixed_reserve）
```
