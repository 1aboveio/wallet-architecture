# 0003 允许退款产生负余额

已结算交易发起退款时，如果商户 `available_balance` 不足，允许退款并产生负余额，后续从新收入中自动抵扣。

## 背景

已结算交易退款从 `available_balance` 扣减。但商户可能在退款前已将 available 提现或使用，导致余额不足。

## 决策

允许退款产生负余额，不拒绝退款。

```
原交易 $100，已结算（T+7），保证金 $5
商户在 T+8 提现 $94 → available = $0
T+10 发起退款 $30

记账:
  借  customer:abc:available:USD   -$30.00   (退款)
  贷  receivable:txn:USD           +$30.00

  借  customer:abc:reserve:USD     -$1.50    (保证金退回)
  贷  customer:abc:available:USD   +$1.50

结果:
  available = $0 - $30 + $1.50 = -$28.50
  reserve   = $5 - $1.50 = $3.50
```

## 负余额抵扣机制

后续收入自动冲抵负余额，优先级：

```
1. 新交易结算 → 先抵扣负余额，剩余入 available
2. 保证金释放 → 先抵扣负余额，剩余入 available
```

示例：
```
available = -$28.50

T+17 新交易结算 $94:
  结算净额 = $94 - $0.94(服务费) - $4.70(保证金) = $88.36
  抵扣负余额: -$28.50 → 归零
  实际入账: $88.36 - $28.50 = $59.86
  available = $59.86

T+97 保证金释放 $3.50:
  无负余额 → 全额入账
  available = $59.86 + $3.50 = $63.36
```

## 理由

1. **买家权益保障** — 不因商户提现行为导致退款被拒
2. **平台可控敞口** — 负余额从后续收入自动追回，平台不额外垫款
3. **保证金兜底** — reserve 余额在 90 天后释放，为负余额提供额外缓冲

## 退款校验规则更新

```
Case 1: 原交易待结算
  校验: 退款金额 ≤ 原交易金额 - 已退款累计
  不需要检查余额

Case 2: 原交易已结算
  校验: 退款金额 ≤ 原交易金额 - 已退款累计
  不拒绝 available 不足（允许负余额）
```
