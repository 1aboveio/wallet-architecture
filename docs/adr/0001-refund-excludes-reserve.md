# 0001 退款可用资金不包含保证金

退款校验时，商户可退款资金仅包含 `available_balance` 和 `pending_balance`，不包含 `reserve_balance`。

## 背景

退款校验规则需要判断商户是否有足够资金覆盖退款请求。早期设计中，商户总资金的计算公式为：

```
商户总资金 = available + pending + reserve
```

这意味着保证金也可以用于退款。但这带来了两个风险：

1. **滥用风险** — 商户可能通过退款将保证金提前取出，规避 90 天冻结期
2. **负余额风险** — 如果保证金被退款消耗，90 天后释放时商户可能没有足够资金，产生大面积负余额

## 决策

退款可用资金公式调整为：

```
商户可退款资金 = available_balance + pending_balance
```

`reserve_balance` 不参与退款可用资金计算。保证金只能在 90 天期满后释放到 `available`，届时才可用于退款或提现。

## 影响

- 商户在保证金冻结期间，可退款金额会低于其"账面总资金"
- 降低平台因退款导致的负余额敞口
- 保证金的风控隔离性更强——专款专用，仅用于覆盖争议/拒付
